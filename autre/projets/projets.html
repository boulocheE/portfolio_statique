Les situations d'apprentissage évaluées (SAE) sont des mises en situation professionnelles. Elles permettent de développer différentes compétences, puis de les démontrer. Les SAE ont différents objectifs. Tout d'abord, ils contribuent au développement de la compétence, et soutient l'apprentissage et la maîtrise des ressources. Enfin, il intègre l'autoévalution par l'étudiant. Ainsi, les SAE permettent une individualisation des apprentissages.

SAE 1.01 - Implémentation d'un besoin client

Problématique professionelle

Développer une application permettant la création, la modification et la consultation des données d'une organisation.

Préconisation d'évaluations

  • La manière dont les fonctionnalités ont été implémenté (nommage, choix des structures de données, modularité...);
  • La complétude des fonctionnalités demandées par le client;
  • La qualité des traces des jeux d'essais permettant la validation de ces fonctionnalités.

Description

Cette SAÉ a pour but de générer un emploi du temps aléatoire, à partir d'une liste d'étudiants.
Cette application génère dans un premier temps les équipes à partir d'une liste d'étudiants. Enfin, elle génèrera les différents plannings.

Langages utilisés

HTML, CSS, Java

Nombre de participants

Groupe de 5 étudiants

SAE 1.02 - Comparaisons d'approches algorithmiques

Problématique professionelle

L'objectif est de pouvoir montrer sa capcité à résoudre des fonctionnalités simples.

Préconisation d'évaluations

  • La démonstration du bon fonctionnnement des différentes versions des différentes fonctions;
  • La cimparaison des temps d'exécution de chaque version.

Description

Ce projet a pour but principal de sensibiliser sur le temps que peut prendre la résolution d'un même problème selon des algorithmes différents.
Pour ce faire, nous avons utilisé dans ce projet des tableaux que nous avons triés à l'aide de trois tris différents : le tri à bulle, le tri par insertion, et le tri par sélection.

Langages utilisés

Java

Nombre de participants

Groupe de 3 étudiants

SAE 1.03 - Installation d'un poste pour le développement

Problématique professionelle

Chaque système d'exploitation à ses avantages et inconvénients, il peut donc être intéressant de mettre en place une machine virtuelle correspondant au besoin de l'entreprise.

Préconisation d'évaluations

  • Rédaction d'un mode opératoire.

Description

Dans notre département informatique, le serveur d'application utilise un système d'exploitation Linux, installée sur une distribution debian 11.
Ainsi, nous avons développé une machine virtuelle permettant de se connnecter au compte que nous utilisons dans notre département, sur notre machine personnelle.

Langages utilisés

Linux

Nombre de participants

Groupe de 3 étudiants

SAE 1.04 - Création d'une base de données

Problématique professionelle

Ce projet a pour but principal de permettre un premier contact avec la formalisation et l'implémentation d'une base de données.
La finalité de ce projet a été de prouver le bon fonctionnement de cette base de données, et d'argumenter l'intérêt que pouvait offrir les différentes requêtes proposées.

Préconisation d'évaluations

  • Qualité de la modélisation;
  • Pertinence du modèle logique de données;
  • Qualité du script de création de la base de données;
  • Respect des règles de programmation (indentation, clarté, commentaires...)
  • Pertinence des requêtes (intérêt pour l'entreprise)

Description

En parant d'un besoin exprimé par un client à travers un modèle logique de données (MLD) existant, et de nouveau besoin à étudier, nous avons créé de nouvelles structures.

Langages utilisés

SQL (PostgreSQL)

Nombre de participants

Groupe de 3 étudiants

SAE 1.05 - Recueil de besoin

Problématique professionelle

Ce projet a pour but d'appréhender les besoins du client et de l'utilisateur à ses différentes étapes (entretien, questionnaire, création de personas...)

Préconisation d'évaluations

  • Identifier les problématiques du client
  • Adopter une démarche proactive, créative et critique
  • Respecter les normes en vigueur
  • Communiquer efficacement avec les différents acteurs d'un projet
  • Sensibiliser à une gestion durable

Description

A partir des données que nous avons pu récolter, nous avons conçu une maquette d'un site Web contenant les plannings pour des oraux.
Nous avons donc développer un générateur d'outil wen écoresponsable pour organiser et afficher les équipes ormées par les étudiants et faciliter l'organisation d'oraux par la génération d'un planning. Ce dernier devait contenir les équipes, les salles, la date et l'heure des différents passages.

Outils utilisés

Canva

Nombre de participants

Groupe de 5 étudiants

SAE 1.06 - Découverte de l'environnement économique et écologique

Problématique professionelle

Ce projet a pour but d'apprhéender les enjeux écologiques et économiques, en montrant l'impact du numérique sur l'environnement.
Ensuite, nous avons réfléchi à une manière de développer qui réduirait les impacts environnementaux.

Nombre de participants

Groupe de 7 étudiants
